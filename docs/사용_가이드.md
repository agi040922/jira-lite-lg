### 🤖 6. AI 기능 (Google Gemini)
**구현 기술:**
- Server Actions: `src/app/actions/ai-complete.ts`
- Google Gemini API (Generative AI)
- AI 캐싱 시스템 (24시간 만료)
- Rate Limiting (분당 10회, 일당 100회)
- PostgreSQL 테이블:
  - `ai_cache` - AI 응답 캐시
  - `ai_rate_limits` - 사용량 제한 추적

**할 수 있는 것:**
- ✅ 이슈 자동 요약
  - 긴 이슈 설명을 3~5줄로 요약
  - 캐싱: 동일 요청 시 DB에서 캐시된 응답 반환 (API 비용 절감)
- ✅ 이슈 설명 AI 생성
  - 제목만 입력하면 상세 설명 자동 생성
  - 예: "로그인 API 구현" → 요구사항, 기술 스택, 체크리스트 자동 작성
- ✅ 댓글 자동 제안
  - 이슈 내용 분석 → 적절한 댓글 제안
- ✅ 이슈 자동 라벨링
  - 이슈 내용 분석 → 관련 라벨 추천
- ✅ 중복 이슈 탐지
  - 유사한 이슈가 있는지 AI가 자동 검색
- ✅ AI 사용량 통계 (`AIStatsWithDB.tsx`)
  - 시간대별 사용량 그래프
  - 기능별 사용 빈도
  - 캐시 히트율 (캐시 활용률)
- ✅ Rate Limiting
  - 분당 최대 10회, 일일 최대 100회 제한
  - 초과 시 "한도 초과" 메시지 표시

**테스트 시나리오:**
1. 이슈 상세 페이지에서 "AI 요약" 버튼 클릭
2. Gemini API 호출 → 이슈 설명 요약
3. 응답이 `ai_cache` 테이블에 저장됨 (24시간 유효)
4. 동일 이슈 재요약 → DB 캐시에서 즉시 반환 (API 호출 X)
5. 대시보드 → AI Stats 확인 → 사용량 그래프, 캐시 히트율 확인

  - 이슈 요약, 설명 생성, 댓글 제안

  - 실시간 온라인 상태 확인

### 2. AI-Native 기능 (Google Gemini)
- **AI 캐싱 시스템**
  - SHA256 해시로 요청 식별
  - 24시간 캐시 유효 (API 비용 절감)
  - 캐시 히트율 모니터링
- **Rate Limiting**
  - 분당 최대 10회, 일일 최대 100회
  - DB 트리거로 사용량 추적
- **AI 기능 5가지**
  - 이슈 요약 (AI_SUMMARY)
  - 설명 자동 생성 (AI_SUGGESTION)
  - 자동 라벨링 (AI_AUTO_LABEL)
  - 중복 탐지 (AI_DUPLICATE)
  - 댓글 요약 (AI_COMMENT_SUMMARY)

### 3. 이메일 발송 (Resend API)
- **팀 초대 이메일**
  - JWT 토큰 기반 초대 링크 (7일 유효)
  - `/api/send-invite` API Route
  - HTML 템플릿 이메일
- **비밀번호 재설정**
  - 1시간 유효 토큰

### 4. Soft Delete 패턴
- **논리적 삭제**
  - `deleted_at` 컬럼 사용 (NULL = 활성, NOT NULL = 삭제)
  - 삭제 시 현재 시간 저장
  - 휴지통에서 복구 가능 (deleted_at → NULL)
- **데이터 보호**
  - 실수로 삭제해도 복구 가능
  - 영구 삭제는 별도 확인 필요

### 5. 데이터베이스 자동화 (22개 트리거/함수)
- **이슈 키 자동 생성** (`generate_issue_key()`)
  - 프로젝트별로 순차 번호 부여 (WEB-1, WEB-2...)
- **프로젝트 기본 상태 생성** (`create_default_project_statuses()`)
  - Backlog, In Progress, Done 3개 자동 생성
- **이슈 히스토리 기록** (`record_issue_history()`)
  - 제목, 상태, 담당자 변경 시 자동 기록
- **자동 알림 발송**
  - `notify_on_assignee_change()` - 담당자 변경 시
  - `notify_on_comment()` - 댓글 작성 시
- **팀 멤버 자동 추가** (`add_owner_to_team()`)
  - 팀 생성 시 owner를 ADMIN으로 자동 추가
- **데이터 제한 검증**
  - `check_project_limit()` - 팀당 최대 15개 프로젝트
  - `check_issue_limit()` - 프로젝트당 최대 200개 이슈
  - `check_subtask_limit()` - 이슈당 최대 10개 서브태스크
  - `check_label_limit()` - 이슈당 최대 5개 라벨

### 6. 파일 처리
- **한글 파일명 지원**
  - `sanitizeFilename.ts` - URL-safe 인코딩
  - `hangul.ts` - 한글 초성/중성/종성 분리
- **Supabase Storage**
  - `editor-images` 버킷 (Public)
  - `videos` 버킷 (Public)
  - 드래그 앤 드롭, 클립보드 붙여넣기

### 7. DB 연동 컴포넌트 (9개)
